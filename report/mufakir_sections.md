## Introduction & Background
The 2014-2016 West African Ebola outbreak was the largest Ebola outbreak since the virus was first discovered in 1976. Caused by the Zaire ebolavirus (EBOV)—a single-stranded, negative-sense RNA virus from the *Filoviridae* family—the epidemic resulted in over 28,000 cases and 11,000 deaths across Guinea, Liberia, and Sierra Leone (Gire et al., 2014; Baize et al., 2014). EBOV expresses seven primary structural proteins (NP, VP35, VP40, GP, VP30, VP24, and L) sequentially from its genome. Quantifying the relative abundance of these transcripts from clinical samples is important for understanding transcription regulation and viral replication dynamics (Shabman et al., 2014).

In this project, we process PRJNA938511, a bulk RNA-seq dataset from the 2014 Makona outbreak variant comprising 356 individual SRA runs totaling approximately 0.83 TB of raw compressed sequence data. Processing data of this magnitude on a local workstation is infeasible due to memory and time constraints. We therefore deployed the pipeline on the Ohio Supercomputer Center's (OSC) Ascend cluster using the SLURM workload manager for parallelized array jobs.

Our pipeline implements a dual-quantification approach: traditional full-alignment (HISAT2 + featureCounts) and pseudo-alignment (Kallisto). The HISAT2 path provides BAM files required for variant calling, while Kallisto provides rapid quantification for cross-validation. This design allows us to verify whether pseudo-alignment reproduces gene-level count matrices generated by full alignment on viral transcriptomes.

## Methods (SRA Extraction, FASTQ, and QC)
Data acquisition for PRJNA938511 comprised fetching 356 SRA runs using the SRA Toolkit (v3.0.2). Retrieval was automated via SLURM array jobs on the OSC Ascend cluster. After downloading, `fasterq-dump` was used to split files into paired-end FASTQ format using `--split-3 -e 10` (10 threads per extraction). Quality assessment was performed with FastQC, generating 576 reports across raw and trimmed reads. Adapter trimming and quality filtering were performed using Trimmomatic in paired-end mode, producing 688 trimmed FASTQ files (344 samples × 2 paired files). Of the 356 SRA runs, 344 yielded usable FASTQ data; 12 failed during download or extraction.

## Methods (HISAT2 Alignment & featureCounts)
Trimmed paired-end reads were aligned to the Ebola virus reference genome (KJ660346.2) using HISAT2 (Kim et al., 2019). Aligned reads were sorted, and PCR duplicates were marked using Picard MarkDuplicates. Post-alignment QC was performed using samtools flagstat and idxstats.

Gene-level quantification was performed using featureCounts (Subread v2.1.1; Liao et al., 2014) in paired-end mode (`-p --countReadPairs`). Of the 344 BAM files, 276 contained valid paired-end reads and were included in the count matrix. The remaining 68 BAMs contained only single-end reads and were excluded from paired-end counting. The final count matrix contains 7 Ebola genes × 276 samples.

## Methods (Kallisto Pseudo-alignment)
The reference FASTA for the Ebola Zaire Makona variant (KJ660346.2) and its GTF annotation were retrieved from NCBI. Transcript sequences were extracted using `gffread`. The Kallisto index was built using Kallisto v0.52.0 with default parameters (k=31). Trimmed paired-end reads for 344 samples were quantified using `kallisto quant` in paired-end mode with 100 bootstraps (`-b 100`).

Each sample's `abundance.tsv` was collected and aggregated to gene level by mapping transcript identifiers to canonical gene names (NP, VP35, VP40, GP, VP30, VP24, L). Output was compiled into `count_matrix_kallisto.csv` and `tpm_matrix_kallisto.csv` using a custom Python script.

## Methods (Variant Calling)
Per-sample variant calling was performed using bcftools (mpileup + call) on 344 BAM files against the KJ660346.2 reference with ploidy set to 1. Variants were filtered (QUAL > 20, DP > 10), normalized, and summarized using bcftools stats. Consensus FASTA sequences were generated for each sample. Samples lacking BAM files (12 total) were skipped.

## Methods (Pipeline Orchestration)
The full pipeline was managed by a coordinator script (`coordinator.sh`) submitted as a SLURM job. This script submits each pipeline step sequentially, waiting for completion before submitting the next, to stay within the SLURM 1,000 MaxSubmitJobsPerUser limit. The coordinator supports `--start-from` and `--stop-at` flags for partial re-execution. All steps use checkpoint files (`.checkpoints/`) to skip previously completed samples on re-runs.

## Results

### Sample Processing Summary
Of the 356 SRA runs in PRJNA938511, 344 (96.6%) were successfully downloaded and converted to FASTQ. Of these, 276 (79.8% of successful) produced valid paired-end BAM alignments used by featureCounts. The Kallisto pseudo-aligner processed all 344 samples. Variant calling completed for 344 samples.

### featureCounts Quantification
The featureCounts count matrix comprises 7 Ebola genes across 276 samples. Total assigned read pairs across all samples: 15,131,200. The median read count per sample was 17 read pairs; the maximum was 3,678,592 (SRR38105630). Of the 276 samples, 177 (64.1%) had detectable Ebola reads (>0 assigned read pairs); 99 samples (35.9%) had zero Ebola reads. This distribution reflects the clinical nature of the samples—these are human blood specimens where Ebola viral RNA constitutes a small fraction of total RNA.

The L gene (RNA-dependent RNA polymerase, the largest gene at ~6.9 kb) accounted for the highest fraction of assigned reads. The featureCounts assignment summary showed that 99.7% of aligned reads were unmapped to Ebola genes, consistent with the fact that these are human blood RNA samples.

### Kallisto Quantification
Kallisto processed 344 samples, averaging under 3 seconds per sample on 4 CPUs. The final matrix contains 7 genes × 344 samples.

### Cross-Validation: featureCounts vs Kallisto
Per-gene total expression correlation between featureCounts and Kallisto across 276 common samples yielded Pearson r = 0.9569. Per-sample total viral load correlation (log₁₀ scale) yielded Pearson r = 0.9978. Both methods produced consistent quantification across the Ebola transcriptome.

### Gene-Gene Correlation
All pairwise Pearson correlations between the 7 Ebola genes were ≥ 0.97 (log₂ counts, 276 samples). This high co-expression is expected for a non-segmented negative-sense RNA virus where all genes are transcribed sequentially from a single promoter.

### DESeq2 Exploratory Analysis
DESeq2 was run in exploratory mode (no sample metadata with condition labels was available). Variance-stabilizing transformation was applied. Outputs include: PCA plot, sample-to-sample distance heatmap, top variable genes heatmap, and expression distribution histogram. The PCA plot shows separation between high-viral-load and low-viral-load samples along PC1.

### Variant Calling
Variant calling with bcftools completed for 344 samples. Per-sample VCF files, filtered variants, and consensus FASTA sequences were generated.

## Discussion
The pipeline processed 344 of 356 SRA runs (96.6%) end-to-end. The 12 failed samples were due to SRA download or extraction failures. Of the 344 aligned samples, 68 contained only single-end reads in the BAM files and were excluded from the paired-end featureCounts analysis, resulting in 276 samples in the final count matrix.

The dual-quantification approach confirmed that Kallisto pseudo-alignment and HISAT2 + featureCounts produce highly concordant results on the Ebola transcriptome (per-sample r = 0.9978). Because EBOV has a linear, non-segmented genome without alternative splicing, Kallisto's k-mer pseudo-alignment is sufficient for accurate gene-level quantification. However, the HISAT2 alignment path remains necessary for generating BAM files required by variant calling.

The gene expression profile—with L as the most abundant transcript—differs from earlier reports where NP typically dominates (Shabman et al., 2014). This may reflect read-length bias (L is the largest gene at ~6.9 kb), differences in library preparation, or the specific clinical stage of the samples in this cohort. The high gene-gene correlations (all r ≥ 0.97) are consistent with the sequential transcription gradient characteristic of non-segmented negative-sense RNA viruses.

The low Ebola detection rate (64.1% of samples with >0 reads, 99.7% of aligned reads unmapped to Ebola) is expected for clinical blood samples. Most RNA in these specimens is human, and viral RNA represents a variable and often small fraction depending on the patient's stage of infection and viral load.

A limitation of this analysis is the absence of sample metadata (clinical condition, time point, outcome). DESeq2 was therefore run in exploratory mode only. The 68 single-end BAM files excluded from featureCounts may represent samples with different library preparation protocols.

## Conclusion
We deployed an end-to-end, checkpoint-aware RNA-seq pipeline on the OSC Ascend HPC cluster, processing 344 of 356 Ebola virus SRA runs from PRJNA938511. The pipeline produced gene count matrices via both featureCounts (276 samples) and Kallisto (344 samples), variant calls for 344 samples, and DESeq2 exploratory analysis. Cross-validation between the two quantification methods yielded Pearson r = 0.9978 at the sample level. The pipeline architecture, including SLURM array jobs with checkpointing and a sequential coordinator, is reproducible and can be applied to other viral RNA-seq datasets.
